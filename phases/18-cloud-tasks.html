Phase 18 — Cloud Tasks

Claude works overnight while you sleep.

What you'll learn

  • Schedule Claude to work overnight while you sleep
  • Use the -wrap and -gm commands for end-of-night and morning handoffs
  • Understand NUTS depth tags — how deep the overnight agent goes
  • Check what Claude accomplished with -cc (cloud check)

Scheduled cloud agents let you give Claude work to do while you're away. You go to bed, Claude plans your next feature. You wake up, the plan is ready to review.

The Concept

Instead of running Claude Code on your computer, you schedule a task that runs in the cloud on a timer. When you wake up, the work is done — sitting on a branch, ready for you to review.

What is a cloud agent?

A cloud agent is Claude running on Anthropic's servers instead of your computer. You schedule a task with a time, and when that time comes, the cloud agent wakes up, does the work, and saves the results to a branch. Your computer doesn't need to be on.

The End-of-Night Handoff

Before you stop for the night, run:

-wrap

This does:

  1. Session recap — what you worked on, what got committed
  2. Shows open todos
  3. Tags 1-2 items with NUTS depth tags for the overnight cloud task
  4. Auto-archives completed items
  5. Commits and pushes docs/TODO.md

NUTS Depth Tags

Tags control how deep the overnight agent goes on each task (the full NUTS flow is taught in Phase 11):

TagWhat Runs OvernightOutput
[PLAN]Plan onlyA plan doc
[PC]Plan + Critique (converge)A refined plan doc
[PCA]Plan + Critique + AuditAn audited plan doc
[BUILD]Full NUTS through codeCode on a feature branch
[AUTO]Full autonuts (all steps)Ready to merge

Recommendation: Start with [PLAN] or [PC]. Building code overnight is less reliable — review plans in the morning and build locally.

The Morning Routine

-gm

Good morning. This command:

  1. Fetches today's daily branch (if it exists)
  2. Reads the overnight summary
  3. Presents: status, plan summaries, critical blockers
  4. Asks what you want to do: pick up planned items, address blockers, or something else

Setting Up a Cloud Task

Use the -cloud command:

-cloud "Plan the user authentication feature" 6am

This schedules a cloud agent to run at 6am, creating a plan doc for user authentication.

Depth Options for -cloud

DepthWhat RunsDefault?
Plan onlyResearch + write plan doc
Plan + CritiquePlan, critique until convergenceDefault
Plan + Critique + AuditAdds audit5 with "what worries you?"
BuildFull NUTS through codeWarn: less reliable in cloud
AutoFull autonutsWarn: recommend local

Checking What Was Done

-cc

Cloud check. Fetches all remote branches, checks which feature branches are ahead of main, reads the overnight briefing, and summarizes what's ready to merge.

Rules

  • Max 2 tagged items per night
  • Tags are removed from TODO after the morning task processes them
  • Start with planning-only depths until you're comfortable

Don't Worry About

  • Cloud task infrastructure — it runs on Anthropic's servers (or use Telegram for real-time remote access)
  • Build-depth overnight tasks — plan + critique is the sweet spot
  • Setting this up on day one — it's a workflow optimization for later
