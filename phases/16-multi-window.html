Phase 16 — Multi-Window

Run multiple Claude sessions — many planners, one builder.

What you'll learn

  • Run multiple Claude sessions at once safely
  • Know the rule — many planners, one builder
  • Understand when files conflict and how to avoid it

You can run multiple Claude Code sessions at the same time. But there's one rule that keeps it from blowing up.

The Rule: Many Planners, One Builder

ActivityTouches Files?How Many Windows?
Planning (-present, -nuts step 1, -c on a plan)No — read-onlyAs many as you want
Building (-build, -autonuts, writing code)Yes — writes filesOnly one at a time

Why: Claude Code reads and writes your actual files. If two windows build simultaneously, they'll edit the same files, create merge conflicts, and cascade into a mess. Planning is read-only, so it's always safe to parallelize.

What is a merge conflict?

A merge conflict happens when two sessions edit the same file at the same time. Git doesn't know which version to keep, so it marks the file as "conflicted" and asks you to resolve it manually. This is why only one session should build at a time — planning sessions don't edit files, so they're always safe to run in parallel.

How to Set It Up

  1. Open multiple terminal tabs or windows
  2. In each one, navigate to your project folder and run claude
  3. Mentally label each: "this one is planning," "this one is building"
  4. When the builder finishes and pushes, another window can start building

Advanced: Git Worktrees

For true parallel building, Claude can create worktrees — isolated copies of your repo where changes don't conflict. Each worktree is a separate folder with its own branch. Two builders can work simultaneously because they're editing different copies.

This is the escape hatch when you genuinely need two builders. But start with the "many planners, one builder" rule — it covers 90% of use cases.

Don't Worry About

  • Running out of terminal windows — your computer can handle many
  • Worktrees on day one — they're an advanced feature
  • Perfect window management — just remember the one rule
