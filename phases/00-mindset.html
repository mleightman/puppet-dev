Phase 0 — Mindset

What vibe coding actually is and why directing AI is a skill.

What Is Vibe Coding?

Vibe coding is a way of building software where you don't write code yourself. You describe what you want in plain English, an AI writes the code, and you verify it works. You're the director. The AI is the developer.

This isn't cheating. It isn't easy. Directing an AI to build something that works requires clarity, patience, and a specific set of skills that this guide teaches you.

The Cast

Think of it like a puppet show. You're the puppet master — you decide what gets built, when, and how to check the work. Claude is the puppet developer — it writes all the code, but only what you tell it to.

What You'll Have by End of Day

If you work through this guide in order, by the end of the day you'll have:

  1. A real app, live on the internet, that you built by directing Claude
  2. A GitHub repository (your code's home)
  3. A CLAUDE.md file that teaches Claude your project every session
  4. A working knowledge of how to plan, critique, and ship features

What You'll Understand (and What You Won't)

You will not understand the code Claude writes. That's fine. Here's what you will understand:

You Will UnderstandYou Won't Understand (And Don't Need To)
Your app's structure (which files do what)The code inside those files
How to describe features clearlyHow to write code that implements them
How to test if something works (by using the app)How to read test files
How to spot when something is brokenHow to fix code by hand
How to deploy your app to the internetHow servers work under the hood

Your job is to verify behavior, not read syntax. "Does the button work?" not "Is line 47 correct?"

What Claude Can't Do

Claude is powerful, but it has limits. Here's each limit and how you work around it:

LimitationWhat It MeansThe Workaround
Can't browse the web live Claude can't visit a URL and read what's there Paste the content, share a screenshot, or use a plugin that gives Claude web access
Can't see your screen Claude doesn't know what your app looks like right now Take a screenshot and share it — Claude reads images
Can't run servers Claude runs commands, not long-running services Railway (Phase 3) runs your server. Claude just writes the code.
Context window has limits Very long sessions make Claude forget earlier work Start fresh sessions. Your CLAUDE.md (Phase 8) keeps context across sessions.
No memory by default Claude doesn't remember what you did last session CLAUDE.md and the memory system (Phase 8) solve this completely.

Why Directing AI Is a Skill

You wouldn't hand someone a hammer and say "build me a house." You'd give them blueprints, check their work at each stage, and course-correct when something's off. That's what you're learning here — not how to swing the hammer, but how to direct the person who does.

The skills you'll build:

  1. Scoping — Defining exactly what to build (and what NOT to build)
  2. Directing — Giving clear instructions that produce the right output
  3. Critiquing — Checking work before it ships
  4. Recovering — Knowing what to do when things go wrong
What is "shipping"?

"Shipping" means making your work available to users. When you push code to GitHub and it auto-deploys to Railway, you've "shipped" it. It's live. People can use it.
