Phase 5 — Terminal Setup

Install your tools and learn the 5 commands that matter.

What you'll learn

  • Open your terminal and navigate to a folder
  • Install Node.js and Git on your computer
  • Use the 5 terminal commands you need for daily work

The terminal is a text-based way to control your computer. Instead of clicking buttons, you type commands. It looks intimidating at first, but you only need about 5 commands for daily use — Claude handles everything else.

Why use the terminal at all?

Claude Code runs in the terminal. It reads your files, writes code, and runs commands — all through text. The terminal is the workshop where the building happens. You don't need to become a terminal expert — just know how to open it, navigate to your project folder, and run a few basic commands.

Open Your Terminal

Mac

  1. Press Cmd + Space to open Spotlight
  2. Type Terminal
  3. Press Enter

You'll see a window with a blinking cursor. That's the terminal. It's ready for commands.

Windows

  1. Press Win + S to open Search
  2. Type Windows Terminal (or PowerShell if Terminal isn't installed)
  3. Press Enter

You'll see a window with a blinking cursor. That's the terminal. It's ready for commands.

Install Your Tools

Mac Setup

Mac needs three things: Homebrew (a package manager), Node.js (runs JavaScript), and Git (tracks code changes).

Step 1: Install Homebrew

Homebrew is like an app store for developer tools. Paste this into your terminal and press Enter:

/bin/bash -c "$(curl -fsSL https://raw.githubusercontent.com/Homebrew/install/HEAD/install.sh)"

This takes a few minutes. Follow any on-screen prompts. When it's done, you'll see "Installation successful."

Step 2: Install Node.js

brew install node

Verify it installed:

node --version

You should see a version number like v20.x.x. If you see a version number, it worked.

Step 3: Install Git

Git usually comes with Mac's developer tools. Run this to trigger the install:

xcode-select --install

A popup will appear asking to install command line developer tools. Click Install and wait (this can take 5-10 minutes).

Verify it installed:

git --version

You should see something like git version 2.x.x.

Windows Setup

Windows needs WSL2 (a Linux environment inside Windows), Node.js, and Git.

Step 1: Install WSL2

What is WSL2?

WSL2 (Windows Subsystem for Linux) lets you run Linux commands on Windows. Most developer tools are built for Linux/Mac first. WSL2 gives you a Linux terminal inside Windows so you can use the same commands as Mac users. Think of it as "Linux inside Windows."

Open PowerShell as Administrator (right-click Start button, select "Terminal (Admin)"), then run:

wsl --install

This installs WSL2 with Ubuntu (a version of Linux). Restart your computer when prompted.

After restarting, Ubuntu will open and ask you to create a username and password. This is your Linux username — pick something simple.

Step 2: Install Windows Terminal (if not already installed)

Windows Terminal is a nicer terminal app. It may already be installed. If not:

  1. Open the Microsoft Store
  2. Search for Windows Terminal
  3. Click Install

Once installed, open Windows Terminal and select the Ubuntu tab (or set it as default).

Step 3: Install Node.js (inside WSL2)

In your Ubuntu terminal, run these commands one at a time:

curl -fsSL https://deb.nodesource.com/setup_20.x | sudo -E bash -
sudo apt-get install -y nodejs

Verify it installed:

node --version

You should see v20.x.x.

Step 4: Git

Git comes pre-installed with WSL2/Ubuntu. Verify:

git --version

You should see git version 2.x.x.

The 5 Commands You Need

These are the only terminal commands you need for daily use. Claude handles everything else.

CommandWhat It DoesWhen You'd Use ItExample
pwd Shows which folder you're currently in When you're lost and need to know where you are pwd → /Users/you/my-first-app
ls Lists all files and folders in the current directory To see what's in a folder ls → index.html package.json node_modules
cd Navigate into a folder To move to your project folder before starting Claude cd my-first-app
git status Shows which files have changed since your last commit Before committing, to see what's been modified git status → shows modified files in red/green
git pull Downloads the latest code from GitHub to your computer At the start of a session, to make sure you have the latest version git pull → Already up to date.
What does the terminal output look like?

When you type a command and press Enter, the terminal shows the result on the next line(s) and then gives you a new prompt (the blinking cursor) for your next command. If nothing shows up and you get a new prompt, the command worked silently — that's normal for many commands.

If you see red text or the word "error," something went wrong. Copy the error message and paste it to Claude — Claude will help you fix it.

Quick Navigation Pattern

At the start of every coding session, you'll do this:

  1. Open your terminal
  2. cd into your project folder: cd Desktop/my-first-app
  3. git pull to get the latest code
  4. Start Claude Code (you'll learn this in Phase 6)

That's the whole routine. Four steps, every time.
