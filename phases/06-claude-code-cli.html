Phase 6 — Claude Code CLI

Install Claude Code and build your first real thing.

What you'll learn

  • Install Claude Code and connect your account
  • Understand how Claude Code differs from web chat
  • Build your first real thing by pasting your scope doc
  • Commit and push your first code to GitHub

Everything before this was setup. This is where you actually direct Claude to build something. You'll install Claude Code, paste your scope doc from Phase 1, and watch Claude build your app.

Prerequisites

  • A Claude Pro or Max subscription at claude.ai (or an Anthropic API key)
  • Node.js installed (Phase 5)
  • A terminal open (you'll navigate to your project folder in step 3 below)

Install Claude Code

  1. Open your terminal
  2. Run the install command: 
    npm install -g @anthropic-ai/claude-code
    What is npm?

    npm is Node.js's package manager — it installs tools and libraries. The command npm install -g means "install this tool globally so it works everywhere on your computer." You run it once to install Claude Code, then forget about npm.

  3. Navigate to your project folder (replace [your-project-name] with whatever you named your repo in Phase 2): 
    cd ~/Desktop/[your-project-name]
    cd /mnt/c/Users/[your-windows-username]/Desktop/[your-project-name]

    In WSL, your Windows Desktop is at /mnt/c/Users/[your-windows-username]/Desktop/. Replace [your-windows-username] with your Windows login name.

  4. Run Claude Code for the first time: 
    claude
  5. Claude will prompt you to authenticate — follow the on-screen instructions to connect your Claude account
  6. You're in. Claude Code is running and ready for instructions.

What's Different from Web Chat

In Phase 1, you used claude.ai (web chat) to plan. Claude Code is fundamentally different:

Featureclaude.ai (Web Chat)Claude Code (Terminal)
Sees your filesNo — only what you pasteYes — reads your entire project
Writes filesNo — shows code in chatYes — creates and edits actual files
Runs commandsNoYes — runs terminal commands, tests, git
Best forPlanning, brainstorming, scope docsBuilding, debugging, shipping

This is the shift from the briefing room to the workshop. Claude can now see your code, touch your code, and run your code.

Your First Real Build

Open the scope doc you saved in Phase 1. Paste it to Claude Code and say:

Here's my project scope. Build me the starter version.

[paste your scope doc here]

Claude will:

  1. Read your scope doc
  2. Create a project structure (folders and files)
  3. Write the initial code
  4. Explain what it built and why

Let it run. Don't interrupt on the first build — let Claude finish, then review what it made.

Understand What Claude Built

After the first build, ask Claude:

Explain the file structure you just created. One sentence per file.

Claude will walk you through each file and what it does. You don't need to understand the code inside — just know which file is responsible for what.

Make Your First Small Change

Pick something tiny — a text change, a color, a label. Tell Claude:

Change the page title from "My App" to "Task Manager"

Watch Claude find the file, make the edit, and confirm. This is the core loop: you describe, Claude builds.

Commit and Push

Now save your work to GitHub:

Commit and push everything we just built

Claude will:

  1. Stage all the new files
  2. Write a commit message
  3. Push to GitHub

If you set up Railway in Phase 3, your app will auto-deploy within a minute. Check your Railway URL — your app is live.

Updating Claude Code

Claude Code updates frequently. If something isn't working or you see unexpected behavior, update first:

npm update -g @anthropic-ai/claude-code

Alternatives to the Terminal CLI

OptionWhat It IsWhen to Use
Terminal CLIclaude in your terminalFull power, recommended for this tutorial
Desktop AppMac/Windows appSame as CLI but with a window — good if the terminal feels intimidating
VS Code ExtensionInside your code editorIf you already use VS Code
JetBrains PluginInside IntelliJ/WebStormIf you already use JetBrains

All of these use the same Claude brain. The terminal CLI is what this tutorial teaches because it has the most features and works everywhere.

Don't Worry About

  • Understanding the code Claude wrote (you'll learn to read structure, not syntax)
  • Making mistakes (Claude can undo, Git can revert)
  • The "right" way to organize files (Claude follows conventions)
