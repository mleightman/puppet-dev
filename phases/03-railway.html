Phase 3 — Railway

Get your app live on the internet.

Right now your code lives on GitHub, but nobody can use it yet. Railway takes your code and runs it on a server so anyone with the link can see your app.

What is a server?

A server is a computer that's always on, always connected to the internet, and always ready to show your app to anyone who visits the URL. Your laptop turns off when you close it — a server doesn't. Railway provides the server. You provide the code.

Note: If your first project is a simple website with no backend (no database, no login, no server logic), you can use GitHub Pages instead of Railway — it's free and simpler. Come back to Railway when your app needs a server.

Create Your Railway Account

  1. Go to railway.app
  2. Click Login
  3. Choose Login with GitHub (use the account you created in Phase 2)
  4. Authorize Railway to access your GitHub repos
  5. You now have a Railway account connected to GitHub

Deploy Your App

  1. From the Railway dashboard, click New Project
  2. Select Deploy from GitHub Repo
  3. Choose the repo you created in Phase 2
  4. Railway will automatically detect your project type and start deploying
  5. Wait 30-90 seconds — you'll see a "Deploying..." status
  6. When it says Active, your app is live

Get Your Public URL

  1. Click on your deployment in the Railway dashboard
  2. Go to Settings → Networking
  3. Click Generate Domain
  4. Railway gives you a URL like your-app-name.up.railway.app
  5. Open that URL in your browser — you should see your app

The Auto-Deploy Loop

Here's the magic: once Railway is connected to your GitHub repo, every time you push new code to GitHub, Railway automatically rebuilds and redeploys your app. You don't have to do anything.

StepWhat HappensHow Long
1. You push code to GitHubGitHub receives your changesInstant
2. Railway detects the pushRailway starts rebuilding your appA few seconds
3. Railway rebuildsYour app is compiled and restarted30-90 seconds
4. New version is liveAnyone visiting the URL sees the updated appDone

Your site does not go down during a redeploy. Railway keeps the old version running until the new one is ready, then switches over instantly.

What is a deployment?

A deployment is the process of uploading your code to a server and making it available to users. Every time you push code to GitHub, Railway creates a new deployment. You can see all your deployments in the Railway dashboard — each one is a snapshot of your app at that moment.

What is a domain?

A domain is the URL people type to visit your app, like your-app.up.railway.app. Railway gives you a free subdomain. If you want a custom domain (like myapp.com), you can buy one later and point it at Railway.

Don't Worry About

  • How servers work at the OS level
  • Docker, containers, or infrastructure
  • Scaling or performance optimization
  • CI/CD pipelines (Railway handles this automatically)
