Phase 15 — When Things Go Wrong

Every beginner hits a wall. Here's how you recover.

Things will break. That's normal. The difference between getting stuck and recovering quickly is knowing the right thing to say to Claude. This phase covers the most common failure modes and exactly how to handle each one.

Deploy Fails

What it looks like: You push code to GitHub, Railway tries to deploy, and the status shows "Failed" or "Crashed." Your live site might show an error or go down.

What to do:

  1. Go to the Railway dashboard and click on your deployment
  2. Click Deploy Logs — this shows the error message
  3. Copy the error message
  4. Tell Claude: This deploy failed. Here's the error: [paste error]
  5. Claude will trace the issue and fix it

Most common causes:

ErrorLikely CauseFix
"DATABASE_URL not found"Missing environment variableAdd it in Railway's Variables tab
"Cannot find module"Missing dependencyTell Claude: "add the missing dependency and push"
"Node version mismatch"Wrong Node.js version on RailwayTell Claude: "set the Node version in package.json"
"Build failed"Code has a syntax errorPaste the full error to Claude

Claude Writes Broken Code

What it looks like: You click a button and nothing happens. Or a page shows an error. Or the layout is completely wrong.

What NOT to do:

  • Don't say "find and fix all the bugs" — Claude will invent problems that don't exist and break working code
  • Don't panic and rewrite everything

What to do:

  1. Describe the specific symptom: "When I click the login button, I see a blank page instead of the form"
  2. Take a screenshot if it's a visual bug and share it with Claude
  3. Use -c to critique what Claude just built before shipping it
  4. If Claude is going in circles, start a fresh session — sometimes Claude gets stuck in a bad mental model
Bad (vague)Good (specific)
"Fix all the bugs""The login button shows a blank page instead of the form"
"It's broken""When I add a task and refresh, the task disappears"
"Something's wrong with the CSS""The sidebar overlaps the content on mobile — see the screenshot"

Something That Worked Stops Working

  1. Run -status — get an honest assessment of the project
  2. Ask: What changed since [feature] last worked?
  3. Run -audit5 to surface what might be wrong
  4. Check recent commits: git log --oneline -10

The Golden Rule

Always describe what you see, not what you think the problem is. "The page shows a white screen after login" is better than "I think the auth token is expired." Let Claude trace the code path. Your job is to report symptoms, not diagnose.
