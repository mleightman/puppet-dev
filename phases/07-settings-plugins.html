Phase 7 — Settings & Plugins

Configure permissions, plugins, and understand costs.

Claude Code has a settings file that controls its behavior — what it's allowed to do, which plugins it has, and how it interacts with you. This phase sets it up properly.

Permissions: Ask vs. Bypass

By default, Claude asks permission before every action — editing files, running commands, etc. This gets tedious fast. Here's the tradeoff:

ModeWhat HappensWhen to Use
Ask every time Claude asks before every file edit, every bash command, every action Your first day, while learning what Claude does
Auto-allow (bypass) Claude runs without asking permission Daily use once you trust the workflow

Recommendation: turn on bypass mode. Here's why:

  • Your CLAUDE.md steering phrases and operating rules are what keep Claude safe — not permission prompts
  • Clicking "allow" on every file read and edit slows you down dramatically
  • If you use the Telegram bot later (Phase 17), bypass is required — you can't approve permissions from your phone
  • You can switch back to ask-every-time instantly — it's one setting change
  • Claude shows every tool call, so you can always review what it did after the fact

How to Enable Bypass Mode

Tell Claude Code:

Set my permissions to auto-allow so you don't ask before every action

Or manually: open ~/.claude/settings.json and set permissions to allow all tools.

Plugins (MCP)

What are plugins / MCP?

MCP (Model Context Protocol) is a system that gives Claude new abilities beyond reading and writing code. Each plugin adds a capability — persistent memory, documentation lookup, browser automation, Telegram messaging. Think of plugins as superpowers you bolt onto Claude.

Plugins are enabled in your ~/.claude/settings.json file. You can ask Claude to help you set them up:

Help me enable the recommended plugins in my settings

Recommended Plugins

PluginWhat It DoesWhy You Want It
claude-mem Persistent memory across sessions Claude remembers what you worked on last time. Searchable history.
compound-engineering Up-to-date library docs, specialized agents, research tools Claude can look up current documentation instead of guessing from training data.
playwright Browser automation Claude can open your app, take screenshots, fill forms, and test UI.
telegram Telegram messaging Control Claude from your phone (Phase 17).

You don't need all of these right now. Start with claude-mem (memory) and add others as you need them.

Claude Code costs real money. Understanding the cost model helps you use it efficiently.

Claude Max $100/month (or $200 for higher limits) For heavy daily use. Higher usage limits, priority access.
API Key Pay per use No monthly fee, but you pay for every token used. Good for light or variable use.

What Costs Tokens

Everything Claude does costs tokens (the unit of AI usage):

  • Every message you send
  • Every file Claude reads
  • Every tool call (edit, bash command, search)
  • Every response Claude generates

Practical tips to manage costs:

  • When a session gets long and Claude starts forgetting or repeating itself, start a fresh session — long contexts burn tokens fast
  • Use /compact to compress the conversation before starting fresh
  • Don't ask Claude to read your entire codebase unless needed — be specific about which files to look at

Don't Worry About

  • Optimizing settings on day one — the defaults work fine
  • Understanding MCP protocol internals
  • Setting up every plugin immediately — add them as you need them
